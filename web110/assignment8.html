SQL exercises

SELECT Statement

  • SELECT * FROM Customers;

Selects only the CustomerID and customer name fields in the Customers table.

WHERE Statement

  • SELECT * FROM Customers WHERE Country='USA';

Selects only CustomerIDs and customer name fields that are located in the USA from the Customers table.

COUNT Statement

  • SELECT COUNT(*) FROM Products;

Counts all the records in the Products table.

INSERT INTO Statment

  • INSERT INTO Customers VALUES ('Cardinal', 'Tom B. Erichsen', 'Skagen 21', 'Stavanger', '4006', 'Norway');

Inserts the values listed for all the columns. This gets inserted into the Customers Table and creates a new record at the bottom of the table.

UPDATE Statement

  • UPDATE Customers SET ContactName = 'Alfred Schmidt', City= 'Frankfurt' WHERE CustomerID = 1;

Updates the ContactName and City for the customer with the CustomerID 1 in the Customers table.

I didn't have any experience with SQL so I learned quite a bit during this unit and exercise. For example, I learned what the basic statements are and do. I had a particularly hard time with the COUNT statement because I couldn't visualize what exactly it was counting. With some digging, I had found that with a * in the parentheses it counts all the records regardless of if they have anything in them or not (null). However, if you put the name of the table in the parentheses then it will NOT count records that are null. The * in the example really threw me off due to this. I decided to think about it in a way that I could better understand it using a language that I already know, javaScript. I thought of it and visualized the statements as If-Then statements and Arrays which, arguably overcomplicates it, but works better for me because I typically visualize the output of code before running it.
